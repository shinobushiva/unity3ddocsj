ShaderLabのシンタックス: Shader

Shaderはシェーダファイルのルートコマンドです。それぞれのファイルは１つ(１つのみ)のシェーダを定義します。シェーダはマテリアルをどのようにレンダリングするかを定義します。

シンタックス

Shader "name" { [Properties] Subshaders [Fallback] } はシェーダを定義します。これは、マテリアルインスペクタの名前(name)以下に表示されます。シェーダはプロパティ(Properties)を定義することができ、マテリアルの設定として使用できます。その後に、サブシェーダと代替のシェーダが続きます。

詳細

プロパティ(Properties)

シェーダはプロパティ(properties)のリストを持つことが出来ます。 シェーダで宣言されたプロパティはマテリアルインスペクタに表示されます。よく使われるプロパティは、オブジェクトの色、テクスチャやシェーダで使用する様々な値です。

サブシェーダと代替(SubShaders & Fallback)

それぞれのシェーダはサブシェーダのリストからなります。少なくとも１つが必要です。シェーダをロードするとき、Unityはサブシェーダのリストを読み、エンドユーザのマシンでサポートされている、最初のものを選びます。サポートされているサブシェーダがない場合、Unityは代替(fallback shader)を使用します。

グラフィックカードは、それごとに異なる機能を持っています。開発者に取って永遠の課題です。ゲームを最新のハードウェアを用いて素晴らしい見栄えにしたいと考えますが、全てのユーザの３％しかプレイできないゲームにしたくはありません。ここでサブシェーダが登場します。１つは、想像できる最高のグラフィック効果を持ったサブシェーダを作り、もう１つは、古いグラフィックカード用のサブシェーダを作ります。これらのサブシェーダでは、効果を遅い方法で実装したり、いくつかの詳細な効果を実装しなかったりします。

例

非常にシンプルなシェーダの例です:








			// colored vertex lighting







Shader "Simple colored lighting" {







    // a single color property







    Properties {







        _Color ("Main Color", Color) = (1,.5,.5,1)







    }







    // define one subshader







    SubShader {







        Pass {







            Material {







                Diffuse [_Color]







            }







            Lighting On







        }







    }







}

このシェーダは色のプロパティ _Color (インスペクタでは Main Color と表示されます) がデフォルト値(1, 0.5, 0.5, 1)で定義されています。次に、１つのサブシェーダが定義されています。このサブシェーダは、頂点ライトをオンにして、基本的なマテリアルを適用する一つのパスPassからなります。
