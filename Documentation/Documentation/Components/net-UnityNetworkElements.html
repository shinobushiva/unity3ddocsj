Unityのネットワーク概要

Unityのネイティブネットワークは前のページで説明されているすべてをサポートしています。Server creation and client connection, sharing data between connected clients, determining which player controls which objects, and punching through network configuration variations are all supported out of the box. このページでは、これらのネットワーキングのUnityでの実装について見ていきます。

 

サーバーの作成

あなたがネットワークゲームをプレイを開始する前に、あなたと通信するコンピュータを決定する必要があります。これを行うには、サーバーを作成する必要があります。これはゲームを実行しているマシンである場合もあれば、ゲームに参加していない専用のマシンにする場合もあります。サーバーを作成するには、スクリプトからNetwork.InitializeServer()を呼び出します。あなたがクライアントとして既存のサーバーに接続する場合は、代わりにNetwork.Connect()を呼び出します。

一般的に、ネットワーククラス（Network class）全体を理解しているとそれが非常に便利であることがわかります。

 

ネットワークビューを用いた通信

ネットワークビューはネットワーク経由でデータを送信するコンポーネントです。ネットワークビューは、ゲームオブジェクトにRPCおよび状態の同期を使用してデータを送信することができます。ネットワークビューを使用する方法次第で、ゲームがどうのような動作を行うかが決定されるでしょう。ネットワークビューにはあまりオプションがありませんが、それはネットワークゲームにおいて非常に重要なものとなるでしょう。

ネットワークビューの使用方法の詳細については、Network View Guide pageとComponent Reference pageを参照してください。

 

リモートプロシージャコール

リモートプロシージャコール（RPC）は、ネットワークビューを含むゲームオブジェクトに添付されているスクリプトで宣言された関数です。ネットワークビューは、RPC関数を含むスクリプトを指してる必要があります。RPC関数は、そのゲームオブジェクト内の任意のスクリプトから呼び出すことができます。

UnityでRPCを使用する方法の詳細については、RPC Details pageをご覧ください。

 

状態の同期

状態の同期は、全てのゲームクライアント間のデータの断続的な共有です。プレイヤーの位置は全てのクライアントについて同期が行われるので、実際はデータがネットワークを介して配信されていてもまるでそれがローカルで制御されているように見えます。ゲームオブジェクト内の状態を同期するには、そのオブジェクトにネットワークビューを追加し、観察対象を指定するだけです。観察データは、その後にゲーム内に全てのクライアントで同期されます。

Unityで状態の同期の使用方法の詳細については、State Synchronization pageをお読みください。

 

Network.Instantiate()

Network.Instantiate()は全てのクライアント上にプレハブをインスタンシエイトするのに自然で簡単な方法です。基本的にInstantiate()と宣言されますが、これは全てのクライアント上でインスタンシエイトを実行します。

内部的なネットワーク。インスタンシエイトは、単純にクライアント上で（または局所的に）実行されるバッファのRPCコールです。それは、全てのクライアント間で正しく同期されているかを確認し、ネットワークビューIDをインスタンシエイトしたプレハブに割り当てます。

詳細についてはネットワークインスタンシエイトのページをご覧ください。

 

NetworkLevelLoad()

共有するデータやクライアントのプレイヤーの状態、およびロードレベルの扱いは圧倒的量になることがあります。Network Level Loadのページでは、このタスクを管理するための有効な事例が紹介されています。

 

マスターサーバー

マスターサーバーは、あなたがゲームにマッチする手助けをします。あなたがサーバーを起動する際、マスターサーバーに接続し、全てのアクティブなサーバーのリストを提供します。

The Master Server is a meeting place for servers and clients where servers are advertised and compatible clients can connect to running games.これは全ての関係者のIPアドレスをいじる必要性を防ぎます。It can even help users host games without them needing to mess with their routers where, under normal circumstances, that would be required. It can help clients bypass the server's firewall and get to private IP addresses which are normally not accessible through the public internet.これは、接続の確立を促進するファシリテーターの助けを借りて行われます。

詳細については、マスターサーバーのページをご覧ください。

 

帯域幅の最小化

ゲームを正常に実行させるため、帯域幅の最小量を使用することは不可欠です。There are different methods for sending data, different techniques for deciding what or when to send and other tricks at your disposal.

帯域幅の使用量を削減するトリックのヒントは、帯域幅の最小化のページに記載しています。

 

ネットワークゲームのデバッグ

Unityには、ネットワークゲームのデバッグに役立つ機能がいくつか搭載されています。

 
  1. ネットワークマネージャーは、全ての着信および発信のネットワークトラフィックのロギングに使用することが出来ます。
  2. インスペクタと階層ビューを使用することで、オブジェクトの生成を追跡し、ビューIDなどを調べることが出来ます。
  3. あなたのマシン上でUnityを２つ起動し、それぞれ異なるプロジェクトを開くことが出来ます。Windowsでは、別のUnityのインスタンスを起動し、プロジェクトウィザードからプロジェクトを開くことにより行うことができます。On Mac OS Xでは、複数のUnityのインスタンスは、ターミナルから開くことが出来、projectPathの引数を使用することが出来ます。
			/Applications/Unity/Unity.app/Contents/MacOS/Unity -projectPath "/Users/MyUser/MyProjectFolder/"
    /Applications/Unity/Unity.app/Contents/MacOS/Unity -projectPath "/Users/MyUser/MyOtherProjectFolder/"

Make sure you make the player run in the background when debugging networking because, for example, if you have two instances running at once, one of them doesn't have focus. これはネットワークループを中断し、望ましく無い結果が発生します。エディタの[編集]→[プロジェクト設定]→[player]または[Application.runInBackground]で有効にすることが出来ます。
