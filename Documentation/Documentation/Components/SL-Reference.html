シェーダーリファレンス(Shader Reference)

Unityのシェーダは次の３つの方法で記述できます：

  • サーフェースシェーダー(surface shaders)として
  • 頂点、フラグメントシェーダー(vertex and fragment shaders)として
  • 固定ファンクションシェーダー(fixed function shaders)として

どのタイプを選択するべきかは、シェーダーチュートリアル(shader tutorial)に記述されています。

どのタイプを選ぶかにかかわらず、シェーダのコードはShaderLabをラップした言語で書かれます。これは、シェーダの構造を組織化するもので、次の様に書かれます：

				Shader "MyShader" {
    Properties {
        _MyTexture ("My Texture", 2D) = "white" { }
        // other properties like colors or vectors go here as well
    }
    SubShader {
        // here goes the 'meat' of your
        //  - surface shader or
        //  - vertex and program shader or
        //  - fixed function shader
    }
    SubShader {
        // here goes a simpler version of the SubShader above than can run on older graphics cards
    }
}

まずは、基本的なShaderLab文法のコンセプトに従って記述することをお勧めします。以下に記述されています。その後、サーフェースシェーダ、頂点・フラグメントシェーダのセクションを読み進めてください。固定ファンクションシェーダはShaderLabのみを利用して記述されるため、ShaderLabのリファレンスを参照すると、より詳しい内容が理解できるでしょう。

以下のリファレンスは、異なるタイプのシェーダに関するサンプルを含みます。サーフェースシェーダのさらなる例が欲しい場合はUnityのビルトインシェーダを参照セクションで見つけられます。Unity Proのスタンダードアセットは頂点・フラグメントシェーダを用いた多くの興味深いイメージ効果を含んでいます。
