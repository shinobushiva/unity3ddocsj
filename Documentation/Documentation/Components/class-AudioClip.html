オーディオクリップ(Audio Clip)

オーディオクリップはオーディオソース(Audio Sources)によって使われるオーディオデータです。Unityはモノラル、ステレオとマルチ（最大８）チャンネルのオーディオアセットをサポートします。Unityは次のオーディオファイルのフォーマットをインポートします: .aif、.wav、.mp3、と.ogg、と次のトラッカーモジュールのファイルフォーマット: .xm、.mod、.it、と.s3m。波形のプレビューがアセットのインポートインスペクタで表示できることを除いて、トラッカーモジュールのアセットはUnityの他のオーディオアセットと同じように動作します。


オーディオクリップのインスペクタ

プロパティ(Properties)

   
Audio Format 特定のフォーマットは音の再生のために使用されます。
Native 大きいサイズのファイルは、高品質です。非常に短いサウンドエフェクトに最適です。
Compressed 小さいサイズのファイルは、低/可変な(variable)品質です。中位の長さのサウンドエフェクトとミュージックに最適です。
3D Sound 有効にした場合、3D空間でサウンドが再生されます。モノラルとステレオの両方とも3Dで再生できます。
Force to mono 有効にした場合、オーディオクリップはシングルチャンネルサウンドにダウンミックスされます。
Load Type オーディオアセットの実行に読み込む方法です。
Decompress on load ロード中にサウンドを解凍します。その場で解凍の実行オーバヘッドを避けるために、サウンド圧縮をより小さくするためにこれを使います。圧縮されたサウンドを解凍する時１０倍以上のメモリを使用します、なので大きいファイルでは使用しないでください。
Compressed in memory メモリ上に圧縮したサウンドを保持し再生中に解凍します。これ小さな実行オーバヘッドを持ちます。（特に、Ogg/Vorbis圧縮ファイル）なので大きいファイルのためにこれを使用します。
Stream from disc ディスクから直接オーディデータを流します。これはメモリのオリジナルサウンドサイズの少量を使用します。音楽または非常に長いトラックのためにこれを使用します。ハードウェアに応じて、一般的なアドバイスはストリームを同時に１－２に到るまでこれを保持することです。
Compression 圧縮量は圧縮クリップに適用されます。ファイルサイズについての統計はスライダの下に見ることができます。ファイルのサイズと配布のニーズを達成するために小さく十分なサイズであっても、十分な再生場所にするためにスライダをドラッグすることを推奨します。
Hardware Decoding (iOS ONLY) iOSデバイス上で圧縮されたオーディオが利用可能です。 より少ないCPUで集中的に復元するためアップルのハードウェアデコーダを使います。詳細はプラットフォームのものをチェックしてください。
Gapless looping

(Android/iOS ONLY) 圧縮した完璧なルーピングオーディオソースファイル（非圧縮のPCMフォーマット上）のループを保持するために使用します。標準MPEGエンコーダはループポイントの周りにサイレンスを取り入れます、そして"クリック"または"ポップ"をわずかに再生します。Unityはあなたのためにスムーズにハンドルします。

オーディオアセットの取り込み(Importing Audio Assets)

Unityは圧縮とネイティブオーディオの両方をサポートします。ファイルの任意のタイプ(MP3/Ogg Vorbisを除く)はネイティブとして初期化インポートされます。ゲームの実行中に圧縮されたオーディオファイルは、CPUによって解凍する必要がありますがファイルのサイズが小さくなります。ストリームがチェックされるならばオーディオは急いで解凍され、あるいはオーディオはロードと同時に完全に解凍されます。ネイティブPCMフォーマット（WAV、AIFF）は、CPU使用の増加なしでより高い忠実性の利点を持ちますが、とても大きいファイルを作成します。モジュールファイル（.mod,.it,.s3m..xm）は非常に少ない容量で高い品質を配信できます。

一般的な経験則として、圧縮オーディオ（またはモジュール）BGMやダイアログのような長いファイルに最適で、非圧縮は短いサウンドエフェクトに優れています。サウンド品質が目立つちょうど前へ圧縮スライダで圧縮量を調整して下さい。

3Dオーディオを使う(Using 3D Audio)

オーディオクリップが3Dサウンドとしてマークされている場合、ゲームワールドの3D空間内でイチをシミュレートして再生します。3Dサウンドは減衰するサウンドとスピーカー全域のパンニングによって距離とサウンドの位置をエミュレートします。モノラルとマルチチャンネルサウンドの両方は3D上に配置できます。複数チャンネルのオーディオについては、スピーカ空間上で別々のチャンネルを広げて外に分けるためにオーディオソース(Audio Source)のスプレッド(spread)オプションを使用して下さい。Unityは3D空間上のコントロールとオーディオの動作を微調整するために様々なオプションを提供しています。オーディオソース(Audio Source)を見てみましょう。

Platform specific details

iOS

モバイルプラットフォーム上で圧縮されたオーディオは、CPUの利用を徹底的に少なくするためにMP3としてエンコードされます。

パフォーマンス上の理由からオーディオクリップはAppleのハードウェアコーデックを使うことで再生できます。有効にするにはオーディオインポーター上の"ハードウェアコーディング"のチェックボックスをチェックして下さい。唯一のハードウェアオーディオストリームは、バックグラウンドのiPodオーディオを含んで一度に解凍できることに注意してください。

もしハードウェアデコーダが利用可能でないならば、解凍はソフトウェアデコーダ（iPhone 3GSか最新のAppleのソフトウェアデコーダはUnityの（FMOD）独自デコーダ上で使用）上でフォールバックします。

Android

モバイルプラットフォーム上で圧縮されたオーディオは、CPUの利用を徹底的に少なくするためにMP3としてエンコードされます。
