ネットワークインスタンシエイト（Network Instantiate）

ネットワークインスタンシエイト（Network Instantiate）はとても簡単な方法ですべてのクライアント上にプレハブをインスタンス化できます。プレイヤーがNetwork.Instantiate()を呼び出したとき、ゲームオブジェクト上で任意の入力コマンドをコントロールします。すべてのセットアップとハンドリングはフードに隠れて実行されます、そのため制御を割り当てるためにそれぞれのプレイヤーは制御すべきゲームオブジェクト上でNetwork.Instantiate()を呼び出す必要があります。

プレハブの生成（Instantiating prefabs）はUnityで最も重要なコンセプトのひとつです。ネットワーク上ですべてのマシン間オブジェクトをインスタンス化するひとつの関数呼び出しをしましょう。ネットワークインスタンシエイトはそのための拡張です。

Network.Instantiate()のための引数リストは次の通りです：

			static function Instantiate (prefab : Object, position : Vector3, rotation : Quaternion, group : int) : Object

Instantiateのように、最初の３つのパラメータを説明すると、設置することになる場所のインスタンス化されるプレハブと位置と回転です。さらにグループは引数として渡されます。通常は、単にグループとして０を渡します。グループナンバーは必要に応じてフィルタリングするためにネットワークのメッセージを一緒にグループ化することができます。

技術詳細（Technical Details）

シーンを超えて、ネットワークインスタンシエイトは単にRPCコール周りに構築された便利な関数です。インスタンス化するためにプレハブの識別子をシリアライズし、シリアライズしたすべてのNetworkViewIDの割り当てをします。位置をシリアライズし、RPCコールを使用してネットワーク経由で送信します。Network.Instantiate()は常にRPCバッファでインスタンシエイトを呼び出します、これは他のRPCコールと同じ方法でバッファから削除ができることを意味します。

本質的には、通常のインスタンス化はすべてのクライアント上で実行し、自動的にバッファリングされます。プレイヤーはインスタンス化が事項された後に加えられ、またインスタンシエイトコールを受信します。

コミュニケーショングループ（Communication Groups）

コミュニケーショングループは通信を区別し、一部のクライアントのため受信・送信を無効にすることが重要です。

例えば、接続された２人のプレイヤーが会うことができずに別の部屋にいるかもしれません。したがって、２人のプレイヤーはアバターの状態や状況に応じて転送する理由はありません。あなたはゲームデータと通信用にひとつのグループを持つことができます。ゲームデータのグループは、レベルのロードするときにブロックされる場合がありますが、チャットグループは開いたままでプレイヤーは通信できます。
