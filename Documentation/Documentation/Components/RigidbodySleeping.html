リジッドボディの非活性化(Rigidbody Sleeping)

例えば、箱が地面にある場合にリジッドボディは非活性化します。非活性化は、物理エンジンにリジッドボディの計算を中断させることで、パフォーマンスを向上します。これにより、基本的に動かない大量のリジッドボディを、シーン中に配置できます。

リジッドボディの非活性化は完全に自動で行われます。リジッドボディがsleepAngularVelocityとsleepVelocityより遅くなったとき、非活性化が始まります。数フレーム経過した後に、完全に非活性となります。リジッドボディが非活性であるとき、衝突判定やシミュレーションは行われません。これによりCPU時間を大きく削減できます。

リジッドボディは次の場合に自動的に活性化します：

  • 他のリジッドボディが非活性なリジッドボディに衝突したとき
  • ジョイント(Joint)で接続された他のリジッドボディが動いているとき
  • リジッドボディのプロパティを編集しているとき
  • 力を加えているとき(adding forces)

リジッドボディを非活性化したいときは、プロパティを変更したり、力を加えたりしないことです。

次の２つの変数でリジッドボディの自動的な非活性化を制御できます。Rigidbody.sleepVelocityとRigidbody.sleepAngularVelocityです。この２つの変数は、物理マネージャ(Physics Manager) (Edit -> Project Settings -> Physics)で定義されるsleepVelocityとsleepAngularVelocityによって初期化されます。

リジッドボディはrigidbody.Sleepを用いて強制的に非活性化できます。これは、新しいレベルをロードしたときにリジッドボディを非活性状態でスタートするのに有効です。

キネマティック(Kinematic)リジッドボディは非活性なリジッドボディを活性化しますが、静的なコライダーは活性化しません。非活性なリジッドボディがあるときに、静的なコライダー(リジッドボディが付加されていないコライダー)をリジッドボディに衝突させたり、リジッドボディの下にある静的なコライダーを引き抜いたりしても、リジッドボディは活性化しません。キネマティックリジッドボディーを引き抜けば、非活性なリジッドボディは活性化し、物理的に正しく計算され動作します。つまり、動かしたい静的なコライダーを使用している場合には、キネマティックリジッドボディコライダーを使用するべきです。
