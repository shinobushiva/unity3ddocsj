Visual Studio C# Integration

この機能で私は何を得ますか？

より洗練されたC#を書く環境です。
スマート補完、コードファイルの変更支援、スマート構文の強調表示などです。

 

ExpressとProの違いはなんですか？

VisualStudio C# 2008はMicrosoftの製品でExpressとProエディションがあります。
Expressエディションは無料で、あなたはここからダウンロードできます。: http://www.microsoft.com/express/vcsharp/
Proエディションは有料で、あなたはここでより詳しい情報を見れます。: http://www.microsoft.com/visualstudio/en-us/products/professional/default.mspx

UnityのVisualStudio統合は２つのコンポーネントがあります :
1) UnityがVisualStudioのプロジェクトファイルを作成し維持します。ExpressとProfesionalで動きます。
2) UnityはあなたがスクリプトかUnity上でエラーをダブルクリックしたとき自動的にVisualStudioを開きます。Profesionalだけです.

 

私はVisual Studio Expressを所持しています、どのように使うのですか？

  • Unity上で、メニューからAssets->Sync VisualStudio Projectを選択します。
  • 新たに作成されたことがわかります。Unityプロジェクトのslnファイルです。(Assetsフォルダからひとつのフォルダに)
  • Visual Studio Expressでファイルを開きます。
  • あなたはこれからすべてのスクリプトファイルを編集でき、Unityに切り替えて使用することができます。
 

私はVisual Studio Professionalを所持しています、どのように使うのですか？

  • Unity上でEdit->Preferencesに移動し、Visual Studioが優先された外部エディタに設定されていることを確認して下さい。
  • プロジェクト上でC#ファイルをダブルクリックして下さい。Visual Studioはあなたのために自動でファイルを開くべきです。
  • あなたはファイルを保存、編集できUnity戻る切り替えができます。
 

Visual Studio C# 2008より他のバージョンで動きますか？

わたしたちVisual Studio 2008とVisual Studio 2010のプレバージョンに対してテストをしました。このリリースの時点で、Visual Studio 2010はまだリリースされていなかったので、リリースされたVisual Studio 2010でうまく動かはわかりません。

 

いくつかの注意点 :

  • VisualStudioはそれ自身C＃コンパイラが付属し、あなたのC#スクリプトにエラーがある場合チェックに使うことができますが、まだUnityはあなたのスクリプトをコンパイルするために独自のC#コンパイラを使います。Visual Studioのコンパイラを使うことはまだ役立ちます、なぜならすべてのエラーかそうでないかのチェックに対してUnityに切り替えなくても良いことを意味するからです。
     
  • Visual StudioのC#コンパイラは現在のUnityのC#コンパイラより機能を持っています。これはいくつかのコード（特に新しいC#の機能）がVisual StudioではエラーでないものがUnityではエラーになることを意味します。
     
  • Unityは自動的にVisual Studioの.slnと.csprojファイルを作成し保持します。誰かがUnity内部でファイルの追加/リネーム/移動/削除をするたびに、Unityは.slnと.csprojファイルを再構築します。あなたは同じようにVisual Studioからソリューションにファイルを追加することができます。Unityは新しいファイルをインポートするでしょう、次にUnityのプロジェクトが再構築するときにこの新しいファイルを含んでプロジェクトを作成するでしょう。
     
  • UnityはAssetServerの更新、またはSVN更新のあとにVisual Studioプロジェクトファイルを再構築しません。あなたは手動でメニューからVisual Studioプロジェクトファイルの再構築ができます。: Assets->Sync VisualStudio Project
